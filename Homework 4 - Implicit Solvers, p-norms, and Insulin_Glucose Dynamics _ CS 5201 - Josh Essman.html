Homework 4 - Implicit Solvers, p-norms, and Insulin/Glucose Dynamics

Purpose

The purpose of this assignment is to continue our exploration of vector analysis and differential equations topics, and to apply modern C++ techniques to implement numerical methods and simulations, in particular:

  • Vector norms and LpLp spaces
  • Numerical integration and differentiation
  • Feedback control theory and possible solutions to the control problem
  • Numerical solutions to “stiff” differential equations
  • Templated functions and objects
  • Operator overloading
  • Interactions between objects of differing classes

Background

When modeling a physical system, it is inevitable that some of the relevant differential equations will be “stiff”, i.e. ill-suited for the explicit Euler method that was implemented in the previous assignment. These equations require implicit methods, e.g., Heun’s method (refer back to your CS3200 textbook if necessary!). Implicit solvers allow complex systems, like the insulin/glucose interactions in the human endocrine and digestive systems, to be accurately simulated. These methods are generally iterative in nature and thus require some form of convergence criteria. The vector norms discussed in class are a common means of establishing the convergence of a system of differential equations. Therefore, you will be responsible for implementing the p-norm for your vector class, which is defined as:

(∑i=0n∣xi∣p)1/p(∑i=0n∣xi∣p)1/p

Now that we have an accurate simulation, we are able to design and test a means of controlling the dynamic system in question. In addition to the insulin/glucose model, you will also implement a closed-loop control system that determines the rate at which to add (inject) insulin given a target blood glucose level (set point). Program Specification

In order to implement an iterative solver for systems of ODEs, you will first need to extend your vector class to calculate a norm. Implement parameterized norm functionality that allows a p-norm to be calculated for any arbitrary integer value of p. Overload the bitwise XOR operator (^), such that vec ^ p returns the p-norm of a vector vec. You should not use std::pow* for this assignment, as the intention is for you to use numerical methods to find integer roots of numbers.

You will then write an implicit ODE solver using the iterative predictor/corrector second (or higher) order Runge-Kutta method of your choice, for example, Heun’s method. As with the explicit Euler method you implemented in the previous assignment, this class should also be templated on the type used to encapsulate the state of the differential equation in order to support solving arbitrary ODEs that are functions of scalars, vectors, or any arbitrary type. This class should have the exact same member functions as your Explicit Euler solver. The only difference will be algorithm used for the calculation of yn+1yn+1, *which should be iterative in nature*.

You are also responsible for the implementation of a discrete-time model of the insulin/glucose dynamics described above.
As with the previous assignment, this should include:

  • Appropriate constructors/destructors/copy functionality
  • function evaluation operator () that “steps” the model, calculating the insulin/glucose concentrations for the next moment in time (taking into account the control parameters for that time step)
  • A means of writing the current state of the model to a std::ostream (stream operator), using the format Glucose: <glucose level>, Insulin: <insulin level>

Finally, you will implement a controller that determines the external insulin input - a PID controller. This class does not need to be templated - you may assume that both your input and control (output) signals are floating-point scalars. This class should include the following:

  • Appropriate constructors/destructors/copy functionality
  • function evaluation operator () that “steps” the controller, calculating the control signal (in our case, insulin level) given an input signal argument (in our case, glucose concentration output from the model)
  • A means of individually setting the control gains KpKp, KiKi, KdKd, using the [] operator - consider the safety of the argument type for this function
  • A means of configuring the controller’s setpoint (in this case, the desired blood glucose concentration) using a reset function, which should accept only the new setpoint.
